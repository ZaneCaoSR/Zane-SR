## 详细问题

### 🟠 严重问题 1: 缺少 API 响应模型

📍 **位置**: `backend/main.py` - 所有 API 端点

❌ **当前代码的问题**:
```python
@app.get("/api/weather/{city_name}")
async def query_weather(city_name: str):
    return await get_weather(city_name)  # 直接返回 dict，无类型约束
```

✅ **建议修改为**:
```python
from pydantic import BaseModel

class WeatherResponse(BaseModel):
    city: str
    temp: str
    weather: str
    # ... 其他字段

@app.get("/api/weather/{city_name}", response_model=WeatherResponse)
async def query_weather(city_name: str):
    return await get_weather(city_name)
```

💡 **延伸说明**: FastAPI 的 `response_model` 可以自动过滤返回字段、验证类型、生成 OpenAPI 文档。当前代码可能泄露敏感字段（如 API Key 相关数据）。

### 🟠 严重问题 2: SQLite 连接未正确关闭风险

📍 **位置**: `backend/database.py:52-70`

❌ **当前代码的问题**:
```python
def add_subscriber(openid: str, city: str = "杭州") -> bool:
    conn = get_connection()
    cursor = conn.cursor()
    # ... 操作
    try:
        cursor.execute(...)
        conn.commit()
        return True
    except sqlite3.IntegrityError:
        # 处理
    finally:
        conn.close()
```

✅ **建议修改为**:
```python
from contextlib import contextmanager

@contextmanager
def get_connection():
    conn = sqlite3.connect(DATABASE_PATH)
    try:
        yield conn
    finally:
        conn.close()

def add_subscriber(openid: str, city: str = "杭州") -> bool:
    with get_connection() as conn:
        cursor = conn.cursor()
        # ... 操作
```

💡 **延伸说明**: 当前代码使用了 try/finally 模式，实际上是安全的。但建议使用 contextmanager 统一管理，更清晰。

### 🟠 严重问题 3: 前端请求缺少统一错误处理

📍 **位置**: `miniapp/utils/request.js:14-34`

❌ **当前代码的问题**:
```javascript
const request = (url, method = 'GET', data = {}) => {
  return new Promise((resolve, reject) => {
    wx.request({
      // ...
      success: (res) => {
        if (res.statusCode >= 200 && res.statusCode < 300) {
          resolve(res.data)
        } else {
          wx.showToast({ title: '请求失败', icon: 'error' })
          reject(new Error(`HTTP ${res.statusCode}`))
        }
      },
      fail: (err) => {
        wx.showToast({ title: '网络错误', icon: 'error' })
        reject(err),
      },
    })
  })
}
```

✅ **建议修改为**:
```javascript
const request = (url, method = 'GET', data = {}) => {
  return new Promise((resolve, reject) => {
    wx.showLoading({ title: '加载中...', mask: true })
    wx.request({
      // ...
      complete: () => wx.hideLoading(),  // 确保 loading 关闭
      success: (res) => {
        if (res.statusCode >= 200 && res.statusCode < 300) {
          resolve(res.data)
        } else {
          wx.showToast({ title: res.data?.detail || '请求失败', icon: 'none' })
          reject(new Error(res.data?.detail || `HTTP ${res.statusCode}`))
        }
      },
      fail: (err) => {
        wx.showToast({ title: '网络连接失败', icon: 'none' })
        reject(err)
      },
    })
  })
}
```

💡 **延伸说明**:
- `fail` 回调时没有隐藏 loading，可能导致界面卡死
- 错误信息应该从响应体中提取，而非固定文本

### 🟠 严重问题 4: setData 频繁调用

📍 **位置**: `miniapp/pages/weather/index.js:270-300`

❌ **当前代码的问题**:
```javascript
loadWeatherForCity(city, index = null) {
  const cities = this.data.cities
  cities[idx].loading = true
  this.setData({ cities: cities, loading: true })  // 第一次 setData

  wx.request({
    success: (res) => {
      // ...
      cities[idx].weatherData = weatherData
      this.setData({ cities: cities })  // 第二次 setData

      cities[idx].weatherTheme = getWeatherTheme(...)
      this.setData({ cities: cities })  // 第三次 setData

      cities[idx].weatherTips = getWeatherTips(...)
      this.setData({ cities: cities })  // 第四次 setData
      // ...
    }
  })
}
```

✅ **建议修改为**:
```javascript
loadWeatherForCity(city, index = null) {
  const cities = this.data.cities
  cities[idx].loading = true
  // 只调用一次 setData，合并所有变化
  this.setData({
    [`cities[${idx}]`]: { ...cities[idx], loading: true },
    loading: true
  })

  wx.request({
    success: (res) => {
      const weatherData = res.data
      // 处理数据...

      // 使用路径语法，只更新需要的字段
      this.setData({
        [`cities[${idx}].weatherData`]: weatherData,
        [`cities[${idx}].weatherTheme`]: getWeatherTheme(res.data.weather),
        [`cities[${idx}].weatherTips`]: getWeatherTips(...),
        [`cities[${idx}].comfort`]: getComfortLevel(...),
        [`cities[${idx}].loaded`]: true,
        [`cities[${idx}].loading`]: false,
        loading: false
      })
    }
  })
}
```

💡 **延伸说明**: 小程序中 setData 是跨线程通信，频繁调用会导致卡顿。每次 Weather 切换调用了 5+ 次 setData，应该合并为 1-2 次。
