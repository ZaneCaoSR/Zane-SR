# 签到积分小程序 需求分析报告

## 执行摘要

本产品定位为一款**积分激励型工具类小程序**，核心功能为每日签到获取积分、积分兑换实物奖品。该项目技术实现难度中等，主要依赖微信登录和订阅消息，无需特殊资质，但需注意实物奖品快递配送的物流对接。MVP 建议聚焦签到核心流程和基础商品库，上线周期约 3~4 周。

---

## 一、产品定位与目标用户

### 目标用户画像

| 用户类型 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 主要用户 | 25~40 岁职场人群 | 希望通过简单行为获得回报，养成早起/签到习惯 |
| 次要用户 | 18~24 岁学生群体 | 喜欢收集积分，对实物奖品敏感，参与抽奖活动 |

### 价值主张

> 为希望养成每日签到习惯的用户，提供**低成本、高回报**的积分获取途径，不同于单纯打卡工具的是，积分可**兑换实物奖品**形成实际利益回馈。

### 现有解决方案不足

- 现有签到类产品多停留在虚拟积分（无法变现）
- 缺乏实物奖品兑换闭环
- 用户签到动力衰减快，缺乏社交刺激

---

## 二、用户故事与使用场景

### 关键用户旅程（主流程）

1. **进入小程序** → 微信扫码或搜索进入
2. **签到** → 点击签到按钮，获得积分
3. **查看积分** → 积分账户实时更新
4. **浏览商品** → 进入兑换商城，浏览可兑换奖品
5. **兑换商品** → 选择商品，消耗积分，下单兑换
6. **填写收货地址** → 填写快递信息，完成兑换

### 典型使用场景

| 场景 | 触发条件 | 用户动作 | 系统响应 |
|-----|---------|---------|---------|
| 晨间签到 | 每天早上起床 | 点击签到 | 弹出签到成功，+10 积分，签到日历更新 |
| 分享炫耀 | 签到后 | 点击分享按钮 | 生成签到卡片，分享到群/好友 |
| 积分不足 | 浏览商品时 | 查看商品所需积分 | 提示"还差 X 积分"，推荐继续签到 |
| 兑换成功 | 积分足够时 | 点击兑换 | 引导填写收货地址，订单状态变为"待发货" |

### 边缘场景

- 断网签到：提示网络异常，缓存签到状态，恢复网络后同步
- 重复签到：同日多次签到无效，提示"今日已签到"
- 积分扣减失败：兑换时积分不足则禁止兑换，分布式事务保障

## 四、技术约束评估

### 微信能力依赖

| 能力 | 必要性 | 备注 |
|------|-------|------|
| wx.login | 必须 | 获取 openid 用于用户识别 |
| 订阅消息 | 推荐 | 用于签到提醒，需用户主动订阅 |
| 微信支付 | 可选 | 若需充值积分需要，个人开发者无法使用 |
| wx.shareAppMessage | 必须 | 分享邀请功能 |

### 包体积评估

- 主包预估体积：约 300~500KB（纯原生开发）
- 可不分包，直接主包上线
- 图片资源建议使用 CDN，商品图控制在 100KB/张

### 性能关键指标

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 首屏加载 | < 2s | 用户登录后直接展示首页签到按钮 |
| setData | < 100KB | 签到状态更新数据量小 |
| API 请求 | < 5 个 | 首页仅需用户信息 + 签到状态 + 积分 |

### 资质要求

- **无需特殊资质**：本产品不涉及支付（积分兑换为商家赠送）、不涉及内容UGC、不涉及医疗/金融
- **企业主体更佳**：个人主体小程序功能无限制，但兑换实物涉及物流建议企业认证

## 五、竞品对比

| 维度 | 本产品 | 竞品A（签到宝） | 竞品B（积分商城） |
|-----|--------|----------------|----------------|
| 核心功能 | 签到+实物兑换 | 签到+虚拟奖励 | 积分任务+兑换 |
| 签到奖励 | 10 积分/天 | 5 积分/天 | 1~5 积分/天 |
| 兑换门槛 | 100 积分起 | 无门槛 | 500 积分起 |
| 实物奖品 | 有（自营） | 无 | 有（第三方） |
| 社交裂变 | 邀请得积分 | 无 | 邀请得积分 |

### 差异化优势

1. **签到即送**：首日签到即送积分，降低用户参与门槛
2. **实物为主**：区别于虚拟优惠券，提供实物奖品吸引力
3. **轻量启动**：功能聚焦签到+兑换，MVP 快速上线

### 规避点

- 避免积分价值标注过高（避免被认定为变相博彩）
- 避免强制分享（违反微信规范）

## 六、风险与建议

### 技术风险

| 风险 | 等级 | 缓解措施 |
|-----|------|---------|
| 微信 API 变更 | 低 | 关注官方公告，及时适配 |
| 签到数据一致性 | 中 | 使用事务 + 乐观锁，防止重复签到 |
| 恶意刷积分 | 中 | IP 限制 + 设备指纹 + 行为检测 |

### 产品风险

| 风险 | 等级 | 缓解措施 |
|-----|------|---------|
| 用户签到动力衰减 | 高 | 连续签到奖励、排行榜、邀请奖励 |
| 奖品库存不足 | 中 | 实物奖品设置每日限量，预警机制 |
| 物流履约成本 | 中 | 与快递公司谈合作价，订单凑够 N 单再发货 |

### 合规风险

| 风险 | 等级 | 缓解措施 |
|-----|------|---------|
| 积分被认定为虚拟货币 | 低 | 积分仅限本小程序使用，不可交易 |
| 奖品为三无产品 | 低 | 采购正规渠道商品，保留发票 |
| 隐私协议缺失 | 中 | 接入前提供隐私协议，说明数据用途 |

### 优先级建议

**MVP 范围**（第一版上线）：
- 微信登录
- 每日签到 + 连续签到奖励
- 积分账户 + 明细
- 商品列表 + 详情
- 积分兑换 + 收货地址
- 订单列表

**第二期**：
- 签到提醒订阅消息
- 邀请好友
- 签到排行榜

**第三期**：
- 抽奖转盘
- 积分充值（企业主体）
